Resource Planning Prototype for Mallette

Goal

Build a working prototype to validate resource planning requirements before deciding buy vs build.

Requirements

RequirementDescription
ResourcesFinancial advisors and accountants with hourly rates
TasksWork items with level of effort (hours), tied to projects
AssignmentOne task per resource at a time
PlanningAssign resources to future tasks based on projected availability
VisibilityStaffing overview showing all assignments over time

Recommended Approach: Microsoft Power Platform

Since Mallette uses Power Platform, building the prototype there provides:

  • Lower development cost (Forrester: 70% reduction vs custom dev)
  • Native Microsoft 365 integration (Teams, Outlook, SharePoint)
  • Dataverse for structured data with relationships
  • Power BI for staffing dashboards and utilization reports
  • Power Automate for notifications and workflow automation
  • Existing licensing - no new infrastructure needed

Power Platform Architecture

Option A: Custom Power Apps Solution (Recommended)

Build a custom Canvas or Model-Driven App on Dataverse:

Dataverse Tables

Resource
├── ResourceId (GUID, Primary Key)
├── Name (Text)
├── Role (Choice: Accountant, Financial Advisor, Tax Specialist, etc.)
├── HourlyRate (Currency)
├── Skills (Multi-select Choice or related table)
├── Email (Text)
└── Office (Lookup to Office table)

Project
├── ProjectId (GUID, Primary Key)
├── Name (Text)
├── Client (Lookup to Account/Contact)
├── Status (Choice: Planning, Active, Completed)
├── StartDate (Date)
└── EndDate (Date)

Task
├── TaskId (GUID, Primary Key)
├── ProjectId (Lookup to Project)
├── Name (Text)
├── EstimatedHours (Number)
├── Status (Choice: Not Started, In Progress, Completed)
└── RequiredSkills (Multi-select or related table)

Assignment
├── AssignmentId (GUID, Primary Key)
├── ResourceId (Lookup to Resource)
├── TaskId (Lookup to Task)
├── StartDate (Date)
├── EndDate (Date, calculated)
├── HoursPerDay (Number, default 8)
└── Status (Choice: Scheduled, Active, Completed)

Option B: Universal Resource Scheduling (URS)

If Mallette has Dynamics 365, Universal Resource Scheduling is built-in:

  • Resource booking with skills/roles matching
  • Prevents overbooking automatically
  • Schedule board visualization
  • Works with Project for the Web

Pros: Already built, enterprise-ready
Cons: May be overkill for prototype, requires D365 license

Option C: Third-Party on Power Platform

PowerRoster (by Velrada) - AppSource solution built on Dataverse for large workforces.

BrightWork 365 - Resource Utilization Power BI Dashboard with 6-week capacity view.

Implementation Plan

Step 1: Set up Dataverse Tables (Day 1)

  • Create Resource, Project, Task, Assignment tables
  • Define relationships and choice fields
  • Set up calculated columns for EndDate

Step 2: Build Model-Driven Apps (Day 1-2)

  • Resource management app
  • Project/Task management app
  • Standard forms and views

Step 3: Build Canvas App for Scheduling (Day 2-3)

  • Visual scheduler interface
  • Drag resources onto tasks
  • Show availability timeline
  • Business rules for conflict prevention

Step 4: Power Automate Flows (Day 3)

  • End date calculation
  • Overlap validation
  • Notifications

Step 5: Staffing Views (Day 4)

  • Table/grid view of all assignments (works without Power BI)
  • Filter by resource, date range, project
  • Optional: Power BI timeline if available

Step 6: Testing & Validation (Day 5)

  • Load sample data
  • Run through verification scenarios

Verification Scenarios

  1. Single assignment: Assign Alice to Task 1, verify end date = StartDate + (40h / 8h/day) = 5 days
  2. Conflict prevention: Try to assign Alice to Task 2 overlapping Task 1 → should fail
  3. Sequential planning: Schedule Task 2 for Alice starting after Task 1 ends
  4. Availability query: Query when Alice is next available
  5. Staffing overview: View all assignments across Feb 2026
  6. Multiple resources: Assign Bob and Alice to parallel tasks

Success Criteria

Prototype is successful if it can:

☐ Create resources with roles and hourly rates
☐ Create projects with tasks and level of effort
☐ Assign resources to tasks with auto-calculated end dates
☐ Prevent double-booking (one task at a time)
☐ Show when resources become available
☐ Schedule future assignments
☐ Display staffing plan overview (timeline view)
